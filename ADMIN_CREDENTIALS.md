## إنشاء حساب مدير يدوياً (إذا لزم الأمر)

### الطريقة 1: استخدام السكريبت

1. تأكد من تشغيل MongoDB
2. من المجلد الرئيسي للمشروع، قم بتشغيل:

```bash
node create-admin.js
```

سيتم إنشاء حساب مدير بالبيانات التالية:

**📧 البريد الإلكتروني:** `admin@greenglass.com`  
**🔑 كلمة المرور:** `admin123`

---

### الطريقة 2: من خلال التسجيل ثم التعديل

1. سجّل حساب جديد من الواجهة (http://localhost:5173/register)
2. افتح MongoDB Compass أو mongo shell
3. ابحث عن قاعدة البيانات `glasscycle`
4. في مجموعة `users`، ابحث عن المستخدم الجديد
5. غيّر `role` من `"user"` إلى `"admin"`

---

### الطريقة 3: من خلال MongoDB مباشرة

1. افتح MongoDB Compass أو mongo shell
2. اذهب إلى قاعدة البيانات `glasscycle`
3. في مجموعة `users`، أنشئ مستند جديد:

```json
{
  "name": "Admin",
  "email": "admin@glasscycle.com",
  "password": "$2a$10$..." // يجب أن يكون مشفراً
  "phone": "0000000000",
  "role": "admin",
  "entityType": "other",
  "isPremium": true,
  "isVerified": true
}
```

**ملاحظة:** كلمة المرور يجب أن تكون مشفرة باستخدام bcrypt. استخدم السكريبت بدلاً من ذلك.

## استكشاف الأخطاء

### خطأ: "Admin user already exists"

**الحل:** 
- استخدم البريد الإلكتروني الموجود: `admin@glasscycle.com`
- أو غيّر البريد الإلكتروني في السكريبت
- أو احذف المستخدم من MongoDB وأعد تشغيل السكريبت

### خطأ: "MongoDB Connection Failed"

**الحل:**
- تأكد من تشغيل MongoDB
- تحقق من `MONGODB_URI` في ملف `.env`

### خطأ: "E11000 duplicate key error collection: glasscycle.users index: username_1"

**الحل:**
هذا الخطأ يحدث بسبب وجود فهرس قديم على حقل `username` غير موجود في النموذج.

**الطريقة 1 (موصى به):**
```bash
npm run fix-index
```
ثم أعد تشغيل:
```bash
npm run create-admin
```

**الطريقة 2: من MongoDB Compass**
1. افتح MongoDB Compass
2. اذهب إلى قاعدة البيانات `glasscycle`
3. افتح مجموعة `users`
4. اذهب إلى تبويب "Indexes"
5. احذف الفهرس `username_1`

**الطريقة 3: من mongo shell**
```javascript
use glasscycle
db.users.dropIndex("username_1")
```

## إنشاء مدير إضافي

لتغيير دور مستخدم موجود إلى مدير:

1. افتح MongoDB Compass
2. ابحث عن المستخدم
3. غيّر `role` إلى `"admin"`

أو استخدم API:

```bash
# بعد تسجيل الدخول كمدير
PUT /api/admin/users/:userId/role
Body: { "role": "admin" }
```
